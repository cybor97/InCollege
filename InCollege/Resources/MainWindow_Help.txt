[Base]В верхнем левом углу любого окна - кнопка меню. Нажав на эту кнопку можно увидеть его меню. Оно же открывается правым кликом по любой не функциональной области окна.
Некоторые окна не содержат меню, просьба не паниковать, если вы его не увидите. Подробнее - "Меню".

Ниже - фильтры. Они позволяют выбрать, какие данные вы хотите отобразить.

Основную часть большинства окон занимает список записей. Все записи в списке можно редактировать или удалять. Для выбора нескольких записей - удерживайте Ctrl и нажимайте на записи. Для выбора всех отображаемых записей - нажмите Ctrl+A.

Ниже - кнопка (+), позволяющая добавлять новые записи.[/Base]

[Files]Файл конфигурации хранит в себе адрес и порт сервера. Файл токена хранит в себе ваш ключ для доступа к серверу. Account.token и Config.json - используются в данный момент.
Копирование этих файлов позволит вам использовать несколько учетных записей или подключений к серверным частям на одном ПК.
ВНИМАНИЕ! Не рекомендуется хранить ключи на рабочем столе, в папке документов и прочих небезопасных местах.[/Files]

[SimpleStatements]С точки зрения данной системы существуют 3 основных вида ведомостей: простая, составная и сводная. 
Простая - содержащая данные только по одной конкретной дисциплине.
Составная - содержащая данные по нескольким дисциплинам.
Сводная - содержит данные по нескольким дисциплинам, даты закрытия и средний бал по дисциплинам.

К простым ведомостям относятся промежуточные и экзаменационные. 
Простые ведомости можно только СОЗДАВАТЬ, но НЕ ГЕНЕРИРОВАТЬ.
Формирование документов из простых ведомостей осуществляется при помощи встроенных шаблонов "Middle" и "Exam".[/SimpleStatements]

[ComplexStatements]С точки зрения данной системы существуют 3 основных вида ведомостей: простая, составная и сводная. 
Простая - содержащая данные только по одной конкретной дисциплине.
Составная - содержащая данные по нескольким дисциплинам.
Сводная - содержит данные по нескольким дисциплинам, даты закрытия и средний бал по дисциплинам.

К составным ведомостям относятся ведомости по окончанию практик, квалификационному экзамену, курсовому проекту и "Прочее". 
К сводным, соответственно, "Сводная".
Составные и сводные можно как СОЗДАВАТЬ, так и ГЕНЕРИРОВАТЬ.
Автоматическое формирование осуществляется по последним результатам ведомостей, являющихся источниками. Результаты различных ведомостей между собой НЕ связаны. 

Формирование документов из составных ведомостей осуществляется при помощи встроенного шаблона "QualificationExam"(Квалификационный экзамен). Для работы с остальными видами необходимо составить шаблоны по примеру существующих.

Формирование документов из сводных ведомостей осуществляется при помощи встроенного шаблона "Total"(Сводная).[/ComplexStatements]

[Accounts]Существуют различные виды аккаунтов.
1. Гость
Может использоваться, например, для абитуриентов. Единственная функция, доступная гостю - переписка.
2. Студент
Студенту доступна переписка и просмотр своей успеваемости.
3. Преподаватель
Преподаватель также может формировать и редактировать ведомости.
4. Зав. отделением
Заведующему отеделением также доступна возможность редактировать данные об учебном заведении.
5. Администратор
Администратор имеет полный доступ ко всем возможностям системы, в том числе редактирование учетных записей других пользователей.

Учетная запись может быть зарегистрирована пользователем или создана администратором. Для использования учетной записи необходимо подтверждение администратора. При смене должности самим пользователем учетная запись становится неподтвержденной и происходит автоматический выход из системы.[/Accounts]
