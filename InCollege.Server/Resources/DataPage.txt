Все методы работают через POST, т.к. кэширование для API не является необходимым.

Для доступа к любому методу нужен токен авторизации, полученный при <a href="/?Auth=1">авторизации</a>.
Передавать его следует параметром token="содержимое_токена".

Существуют такие методы:

<ol>
 <li>GetRange</li>
 Получает диапазон данных. Параметры:
 <ul>
 <li>table - какие данные нужны</li>
 <li>skipRecords - сколько записей необходимо пропустить</li>
 <li>countRecords - сколько записей запрашивается с 0-й либо пропущенной</li>
 <li>column - для какого столбца запрашиваются данные(если только 1)</li>
 <li>fixedString - являются ли параметры whereParams фиксированными(1 или 0). Фиксированный - ТОЧНО РАВНО, не фиксированный - СОДЕРЖИТ</li>
 <li>justCount - запрашивать только количество</li>
 <li>reverse - "перевернуть" результаты</li>
 <li>orAll - для параметров используется логика ИЛИ вместо стандартной И</li>
 <li>whereНазваниеПоля - поле, по которому осуществляется выборка. Может быть неограниченное количество, но названия не могут повторяться</li>
 </ul>

 <li>Save</li>
 Сохраняет данные. Параметры:
 <ul>
 <li>table - что сохранять</li>
 <li>fieldНазваниеПоля - поля для сохранения</li>
 </ul>

 <li>Remove</li>
 Удаляет данные. Параметры:
 <ul>
 <li>table - что удалять</li>
 <li>id - ID записи, которую необходимо удалить</li>
 </ul>

 <li>RemoveWhere</li>
 То же, что и Remove, но вместо id, whereParams, как в GetRange

 <li>Chat</li>
 Обеспечивает взаимодействие пользователей. Параметры:
  <ul>
   <li>mode - режим. Что требуется от метода. Может быть  CheckMessages, Conversation, Friends и Send</li>
   CheckMessages - проверяет наличие сообщений, возвращает их количество.
   Conversation - получает "беседу" - список сообщений, отправленных либо полученных от заданного пользователя
   Friends - получает список "друзей" - пользователей, с которыми вы вели переписку
   Send - отправляет сообщение
   <li>partnerID - ID пользователя, с которым ведется переписка</li>
   <li>Параметры-поля класса "Сообщение". Да, без field в начале. И без FromID.</li>
  </ul>


 <li>Disconnect</li>
 Прерывает соединение
